# CONVOTIS Portfolio & Solution Taxonomy

## Purpose

This taxonomy defines how CONVOTIS solutions, services and campaigns should be categorized across the organization.

The purpose is to:

- Create a consistent structure across Marketing, Sales and Delivery
- Standardize campaign creation
- Support naming conventions
- Improve reporting and performance analysis
- Enable AI-driven campaign planning
- Support audience and offer mapping

All campaigns, assets, audiences, landing pages and HubSpot campaigns should be mapped to this taxonomy.

# Campaign Mapping Rules

Every campaign should be mapped to:

1. Business Unit
2. Solution Cluster
3. Primary Solution
4. Target Segment
5. Buyer Journey Stage
6. Offer

Example:

Business Unit: CS

Cluster: CYB

Solution: Information Security Assessment

Segment: Manufacturing

Stage: Consideration

Offer: Webinar

Campaign Code:

CS_CYB_CON_DE_Manufacturing_Q2-2026_WEB_VID

---

# Key Principle

The taxonomy should be used consistently across:

- LinkedIn Campaigns
- HubSpot Campaigns
- Clay Audiences
- Landing Pages
- Workflows
- Reporting
- AI Campaign Planning

A campaign should never exist without:

- A defined Business Unit
- A defined Solution Cluster
- A defined Primary Solution
- A defined Target Segment
- A defined Offer

If Digital Media Brand activation is enabled, at least one expert knowledge source should also be identified.
